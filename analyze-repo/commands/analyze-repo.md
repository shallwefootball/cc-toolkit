---

GitHub 레포지토리를 분석하는 작업을 수행합니다. 다음 단계를 따르세요:

1. **레포지토리 정보 추출**
   - 사용자가 제공한 GitHub URL에서 owner와 repo name 파싱

2. **레포지토리 클론** `~/Documents/clone/[repo-name]` 경로에
   - 디렉토리가 이미 존재하면 덮어쓸지, 기존 것을 분석할지 사용자에게 물어보기

3. **레포지토리 메타데이터 수집** (git 및 GitHub CLI 사용):
   - GitHub Stars 개수
   - Fork 개수
   - 라이선스 정보
   - 마지막 커밋 날짜
   - 레포지토리 설명

4. **코드베이스 분석**을 철저히 수행:
   - 프로젝트 구조 (주요 디렉토리와 용도)
   - 기술 스택 (package.json, requirements.txt, go.mod, Cargo.toml 등)
   - 핵심 파일과 진입점
   - 주요 의존성
   - 아키텍처 패턴과 코드 구조
   - API 엔드포인트 (있는 경우)
   - 데이터베이스 스키마 (발견되는 경우)
   - 설정 파일들

5. **`REPO-ANALYSIS.md` 생성** (클론된 레포지토리 루트에):

```markdown
# [레포지토리 이름] 분석

## 레포지토리 정보
- **GitHub URL**: [url]
- **설명**: [GitHub에서 가져온 설명]
- **Stars**: ⭐ [개수]
- **Forks**: 🍴 [개수]
- **라이선스**: [라이선스]
- **마지막 커밋**: 📅 [날짜]

## 개요
[이 프로젝트가 무엇을 하는지 간단히 설명]

## 기술 스택
[주요 기술, 프레임워크, 언어 나열]

## 프로젝트 구조
[디렉토리 구조와 설명]

## 핵심 컴포넌트
[주요 파일, 모듈, 그리고 각각의 역할]

## 의존성
[주요 의존성 라이브러리와 사용 목적]

## 아키텍처
[디자인 패턴, 아키텍처 결정사항, 데이터 흐름]

## 진입점
[프로젝트 실행/사용 방법]

## 추가 메모
[분석 중 발견한 기타 관련 정보]
```

6. **최종 결과 출력**:
   - 레포를 어디에 클론했는지 확인
   - 분석 파일을 어디에 생성했는지 확인
   - 발견한 내용 간단히 요약

코드베이스 탐색은 Task 도구를 subagent_type='Explore'로 사용하여 철저히 수행하세요.
