코드 작업 완료 후 문서를 업데이트하고, 코드와 문서가 일치하는지 꼼꼼하게 확인해줘.

## 실행 단계

1. **변경사항 파악**
   ```bash
   # 커밋되지 않은 변경사항 확인
   git diff --name-only
   git diff --staged --name-only

   # 최근 커밋 변경사항도 확인 (아직 push 안 했을 수 있음)
   git log --oneline -5
   git diff HEAD~1 --name-only
   ```

2. **문서 파일 찾기**
   프로젝트에서 문서 파일들을 찾아:
   - `README.md`
   - `docs/` 폴더
   - `CHANGELOG.md`
   - `*.md` 파일들
   - 코드 내 주석/docstring

3. **변경된 코드 분석**
   - 새로 추가된 기능
   - 수정된 API/인터페이스
   - 삭제된 기능
   - 설정 변경사항

4. **문서-코드 불일치 체크**
   꼼꼼하게 확인할 것:
   - [ ] 새 기능이 문서에 있는가?
   - [ ] 변경된 API가 문서에 반영되었는가?
   - [ ] 삭제된 기능이 문서에서도 제거되었는가?
   - [ ] 예제 코드가 현재 코드와 일치하는가?
   - [ ] 설치/사용 방법이 최신인가?
   - [ ] 버전 정보가 맞는가?

5. **문서 업데이트**
   불일치하는 부분을 수정:
   - README 업데이트
   - API 문서 수정
   - 예제 코드 수정
   - CHANGELOG 추가 (있다면)

6. **코드 수정 (필요시)**
   문서를 보면서 코드도 확인:
   - 문서와 맞지 않는 코드 수정
   - 누락된 주석 추가
   - docstring 업데이트

7. **최종 확인**
   ```bash
   git diff
   ```
   변경된 내용을 보여주고, 사용자에게 확인받기.

## 출력 형식

```markdown
## 🔄 코드-문서 동기화 결과

### 변경된 코드 파일
- file1.ts (새 기능 추가)
- file2.ts (API 변경)

### 업데이트한 문서
- README.md: 새 기능 설명 추가
- docs/api.md: API 변경사항 반영

### 수정한 코드
- file3.ts: 문서와 불일치하던 부분 수정

### 확인 필요
- [ ] 추가 검토가 필요한 항목들
```

## 주의사항

- **세세하게 확인**: 작은 불일치도 놓치지 말 것
- **예제 코드**: 실제로 동작하는지 확인
- **버전**: package.json, plugin.json 등 버전 정보 체크
- **링크**: 문서 내 링크가 유효한지 확인

지금 바로 현재 프로젝트의 변경사항을 분석하고 문서를 동기화해줘.
